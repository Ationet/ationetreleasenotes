# ATIONET Console (Web Portal) - v 2.6.8250

## PROD Deployment (Jul 29, 2026)

## NEW FEATURES

### Vehicle/Driver Quota Rule for Networks

A new rule type, **Vehicle/Driver Quota**, has been introduced and is available exclusively for configuration at the **Network** level.
This rule can be assigned to a **Company, Company Contract, and/or Program**, allowing administrators to define the maximum amount of fuel that each vehicle or driver can consume during a specified period, either by fuel volume or monetary value.
This enhancement simplifies quota management by eliminating the need to create individual quota rules for each vehicle or driver.

### Transaction Limit Rule for Networks

**Transaction Limit rules**, previously available only at the Company level, can now also be configured at the **Network** level.
These rules can be assigned to a **Company, Company Contract, and/or Program**, allowing administrators to define the maximum fuel volume or monetary amount permitted for each transaction.

### Company Contract Balance Mode Restriction

Starting with this release, the **Balance Mode** of a Company Contract can no longer be changed between Dispersed and Non-Dispersed after the contract has been created.

## FIXES AND IMPROVEMENTS

The following issues have been resolved:
- Fixed an issue where the PIN Digit Limit field was truncated when editing an Identifier Model.
- Fixed an issue in the Dispersions screen where records were displayed after selecting the Code column without first applying the required filter.
- Fixed an issue that prevented exporting the Identifier Requests report to Excel.
- Fixed the “User not allowed to perform this action” error when printing vouchers.

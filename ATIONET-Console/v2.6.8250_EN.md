# ATIONET Console (Web Portal) - v 2.6.8250

## PROD Deployment (Jul 29, 2026)

### New Vehicle/Driver Quota Rule

A new rule type called "Vehicle/Driver Quota" is now available. This rule can only be configured as a Network Admin and can be applied to a Company, Contract, and/or Program.

These rules define the amount of fuel, either by volume or monetary value, that each vehicle or driver is allowed to consume within the configured period.

This eliminates the need to create multiple individual rules for each vehicle or driver.

### Network Transaction Limit Rule

Transaction Limit rules, previously available only at the Company level, can now also be configured at the Network level and be applied to a Company, Contract, and/or Program.

These rules restrict the maximum amount of fuel, either by volume or monetary value, that can be sold in a single transaction.

### Company Contracts: Balance Mode Restriction

Starting with this version, it is no longer possible to change the balance mode of an existing Company Contract from "Dispersed" to "Non-Dispersed", or vice versa. This restriction prevents inconsistent balance management for existing contracts.

### Fixed Issues and Improvements

The following issues have been resolved:

* Fixed an issue where the "PIN Digit Limit" field was truncated when editing an Identification Model.
* Fixed an issue where Dispersion records were displayed when selecting the Code column without first applying the required filter.
* Fixed an issue when exporting the Identification Requests report to Excel.
* Fixed the "User not allowed to perform this action" error when printing vouchers.

# ATIONET Company Portal - v1.148.3

## PROD Deployment (Jun 17, 2026)

## New Bulk Actions for Vehicles and Drivers

This release introduces new **Bulk Actions** functionalities that can be applied to vehicles in a mass operation.

<img width="149" height="290" alt="image" src="https://github.com/user-attachments/assets/febc39a8-7c32-4b3b-b42f-67f1211fac94" />
<br>

The new bulk actions are:

- **Assign Rule:**
  
  This option allows a previously created rule to be assigned to a group of vehicles.

- **Resend Identification:**
  
  This option allows users to resend an email containing the identification QR code as an attachment, facilitating the mass distribution of credentials. This feature is available if the identification type supports this workflow.

- **Assign Status:**
  
  This option allows users to mark vehicles as **Operational** or **Obsolete**. If marked as Obsolete, the vehicle will no longer be able to perform transactions.

### Individual Action: Assign Identification to Vehicles and Drivers

<img width="149" height="185" alt="image" src="https://github.com/user-attachments/assets/9ad47441-d9f4-4a64-acf9-4cd3173e0743" />

This release includes an individual identification assignment feature within the Vehicles and Drivers module. Unlike bulk actions, this operation is performed on a single entity at a time and can be accessed from the **Options** column in the grid.

<img width="594" height="445" alt="image" src="https://github.com/user-attachments/assets/b6955ba2-4a71-4db4-89e0-8eb89dce6aa0" />

### Individual Action: Lost Identification

<img width="149" height="220" alt="image" src="https://github.com/user-attachments/assets/e4bb27aa-3517-4ca3-9aa7-30c5e67ddd34" />

A new **Lost Identification** functionality has been added, allowing users to mark one or more identifications as unusable within the system. This process records the reason and removes the operational association between the identification and the entity to which it was assigned. The feature is available for both vehicles and drivers through the **Options** column in the grid.

<img width="370" height="329" alt="image" src="https://github.com/user-attachments/assets/d5f2994e-e50c-4366-b490-055d70cb9875" />

To mark an identification as lost, selecting a reason is mandatory. If **Other** is selected, an additional comment describing the situation is also required.

### Individual Action: Identification Renewal

<img width="149" height="220" alt="image" src="https://github.com/user-attachments/assets/7b072bc7-358e-4764-965f-fc052207a66c" />

The vehicle and driver identification renewal feature is also available, allowing users to generate a replacement request by specifying a renewal reason without manually deactivating the current identification.

### Hologram Requests

When requesting or renewing identifications, if the program requires a Hologram, an additional option will now be displayed allowing the user to select the hologram model to be used.

## Fixed Issues

The following issues have been resolved:

- Various translation issues.
- Autocomplete errors in Widgets.
- Display issue with the expand/collapse icon used to show details for all vouchers when refreshing the module without collapsing the information.
- The system did not refresh the selected record data on the main screen after an import operation.
- During vehicle imports, the system updated the file with an incorrect name and without displaying an error message.
- The system reported a successful assignment even when the identification assignment action was not executed.
- Pagination and quantity filter inconsistencies in Voucher Administration.
- Clearing the fleet identification filter also removed placeholder text from the filters.
- Disconnection issues when performing actions in the **Site Transactions** widget.
- Incorrect data entry in the **Volume by Product** widget made it difficult to re-enter the data correctly.
- The **Site Transactions** widget did not allow further interaction after using filters.
- Individual action options were displayed outside the grid when only a few records were present.
